# Use Cases: CRM Task Workflow

Сквозной сценарий работы с CRM-задачами через веб-интерфейс агента.

## Подготовка

1. Убедиться, что сервер запущен: **http://localhost:8081**
2. Открыть браузер, перейти по адресу выше
3. Нажать кнопку **«+ Новый чат»** — откроется новая сессия
4. Все 5 шагов выполняются **в одном чате** (не создавать новый чат между шагами!)

---

## Шаг 1 — Получить список задач из CRM

**Что написать в чат:**
> Покажи все задачи из CRM

**Нажать:** кнопку «Отправить» (или Enter)

**Ожидаемый результат:**
Агент вызовет инструмент `get_tasks()` и вернёт таблицу со всеми задачами:
- Колонки: ID, заголовок, исполнитель, статус, приоритет, дата создания
- Задачи в статусах NEW, IN_PROCESS и DONE

---

## Шаг 2 — Взять задачу в статусе NEW

**Что написать в чат:**
> Покажи задачи в статусе NEW и выбери первую с наивысшим приоритетом

**Нажать:** «Отправить»

**Ожидаемый результат:**
Агент вызовет `get_tasks(status="NEW")` и выдаст отфильтрованный список.
Затем укажет конкретную задачу с приоритетом HIGH (например: «Связаться с клиентом Иванов»).

> **Запомни название выбранной задачи** — оно понадобится на шагах 3 и 4.

---

## Шаг 3 — Найти документацию по задаче через RAG

**Что написать в чат** (подставь реальное название задачи из шага 2):
> Найди в документации информацию, которая поможет выполнить задачу «Связаться с клиентом Иванов»

**Нажать:** «Отправить»

**Ожидаемый результат:**
Система автоматически выполнит RAG-поиск по базе знаний и найдёт релевантные чанки.
Агент вернёт пошаговую инструкцию из `crm_guide.md` — например, для задачи «Связаться с клиентом»:
- Изучить карточку клиента
- Подготовить повестку звонка
- Зафиксировать итог переговоров
- и т.д.

---

## Шаг 4 — Сменить статус задачи на IN_PROCESS

**Что написать в чат** (подставь реальное название задачи из шага 2):
> Возьми задачу «Связаться с клиентом Иванов» в работу — смени её статус на IN_PROCESS

**Нажать:** «Отправить»

**Ожидаемый результат:**
Агент вызовет `update_status(task_id, "IN_PROCESS")` и вернёт подтверждение:
- Название задачи
- Новый статус: **IN_PROCESS**
- ID задачи

---

## Шаг 5 — Проверить, что статус изменился

**Что написать в чат:**
> Покажи снова список всех задач и убедись, что статус задачи изменился

**Нажать:** «Отправить»

**Ожидаемый результат:**
Агент снова вызовет `get_tasks()` и вернёт обновлённую таблицу.
Задача «Связаться с клиентом Иванов» теперь отображается со статусом **IN_PROCESS** вместо NEW.

---

## Итоговый сценарий целиком (краткая версия)

| Шаг | Сообщение в чат | Инструмент MCP |
|-----|----------------|----------------|
| 1 | `Покажи все задачи из CRM` | `get_tasks()` |
| 2 | `Покажи задачи в статусе NEW и выбери первую с наивысшим приоритетом` | `get_tasks(status="NEW")` |
| 3 | `Найди в документации информацию, которая поможет выполнить задачу «...»` | RAG-поиск (авто) |
| 4 | `Возьми задачу «...» в работу — смени её статус на IN_PROCESS` | `update_status(id, "IN_PROCESS")` |
| 5 | `Покажи снова список всех задач и убедись, что статус задачи изменился` | `get_tasks()` |

---

## Технические детали (для справки)

- **Сервер:** `llm_agent_v2/web/api_server.py` на порту 8081
- **MCP-инструменты:** `crm_server.py` (create_task, get_tasks, update_status, delete_task)
- **RAG-индекс:** `llm_agent_v2/rag_index/` — проиндексированы все `*.md` из `llm_agent_v2/`
- **Документ по CRM:** `llm_agent_v2/support_docs/crm_guide.md`
- **Формат данных задачи:** `{ id, title, description, assignee, status, priority, created_at }`
