# README: Клиент для RouterAI API на Go

Данная программа представляет собой простой консольный клиент для отправки запросов к API сервиса [RouterAI](https://routerai.ru) с использованием модели `deepseek/deepseek-v3.2`. Она считывает сообщение пользователя из стандартного ввода, отправляет его в API и выводит полученный ответ.

## Требования

- Установленный Go (версия 1.16 или выше)
- API-ключ сервиса RouterAI (получить можно на [routerai.ru](https://routerai.ru))

## Установка

1. Клонируйте репозиторий или создайте новую директорию для проекта:
   ```bash
   git clone <url-репозитория>
   cd <директория-проекта>
   ```

2. Инициализируйте модуль Go и установите зависимости:
   ```bash
   go mod init routerai-client
   go get github.com/sashabaranov/go-openai
   ```

3. Скомпилируйте программу:
   ```bash
   go build -o routerai-client main.go
   ```

## Настройка

Перед запуском необходимо установить переменную окружения `ROUTERAPI_API_KEY`, содержащую ваш API-ключ.

**Linux / macOS:**
```bash
export ROUTERAPI_API_KEY="ваш_ключ"
```

**Windows (cmd):**
```cmd
set ROUTERAPI_API_KEY=ваш_ключ
```

**Windows (PowerShell):**
```powershell
$env:ROUTERAPI_API_KEY="ваш_ключ"
```

## Запуск

Запустите программу:
```bash
./routerai-client
```

После запуска введите ваш запрос и нажмите Enter. Программа отправит его в API и выведет ответ.

## Пример использования

```
Введите ваш запрос: Напиши кратко о погоде в Москве
Ответ: Погода в Москве сегодня переменная облачность, температура +5°C...
```

## Примечания

- Базовый URL API задан в коде как `https://routerai.ru/api/v1`. При необходимости его можно изменить.
- Модель, используемая по умолчанию: `deepseek/deepseek-v3.2`. Чтобы использовать другую модель, измените поле `Model` в структуре `ChatCompletionRequest`.
- Программа проверяет наличие API-ключа и непустого сообщения. При ошибках выводится сообщение в лог.
- Для работы требуется подключение к интернету и доступность API.
