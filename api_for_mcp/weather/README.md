# Weather API

FastAPI-сервис с погодными данными для 35 российских городов. Все эндпоинты (кроме `/`) защищены авторизацией через заголовок `X-API-Key`.

## Настройка

Создайте файл `.env` в директории `api_for_mcp/weather/`:

```
WEATHER_API_KEY=secret-token
```

Если `.env` отсутствует, используется значение по умолчанию `secret-token`.

## Запуск

```bash
cd api_for_mcp/weather
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Сервис запустится на `http://localhost:8000`.

## Авторизация

Все эндпоинты `/weather/{city}` и `/cities` требуют заголовок `X-API-Key`.

**Без ключа — `401 Unauthorized`:**
```bash
curl http://localhost:8000/weather/москва
# {"detail":"Неверный или отсутствующий токен"}
```

**С ключом:**
```bash
curl -H "X-API-Key: secret-token" http://localhost:8000/weather/москва
```

## Эндпоинты

### `GET /weather/{city_name}` 🔒
Возвращает погоду для указанного города. Поиск регистронезависимый.

```bash
curl -H "X-API-Key: secret-token" http://localhost:8000/weather/Москва
curl -H "X-API-Key: secret-token" http://localhost:8000/weather/санкт-петербург
```

**Пример ответа:**
```json
{
  "city": "Москва",
  "temperature": -3,
  "condition": "Облачно, небольшой снег",
  "humidity": 80,
  "wind_speed": 5,
  "wind_direction": "С"
}
```

**Ошибки:**
```json
// 401 — неверный или отсутствующий ключ
{"detail": "Неверный или отсутствующий токен"}

// 404 — город не найден
{"detail": "Город 'Xxx' не найден. Доступные города: Москва, Санкт-Петербург, ..."}
```

### `GET /cities` 🔒
Возвращает список всех доступных городов.

```bash
curl -H "X-API-Key: secret-token" http://localhost:8000/cities
```

### `GET /`
Health check — информация о сервисе (авторизация не требуется).

```bash
curl http://localhost:8000/
```
