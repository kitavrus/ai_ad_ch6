2. **Ресторан (Restaurant):**  
   - Управляет меню, принимает заказы, обновляет статус готовности блюд.  
3. **Курьер (Delivery Driver):**  
   - Получает заказы, обновляет статус доставки, взаимодействует с системой для маршрутизации.  
4. **Платежный шлюз (Payment Gateway):**  
   - Обрабатывает платежи, возвращает статус транзакции.  
5. **Сервис карт (Mapping Service):**  
   - Предоставляет геолокационные данные, маршруты, расстояния.  
6. **Система поддержки (Customer Support):**  
   - Обрабатывает жалобы, возвращает статусы заказов, взаимодействует с пользователями.  

#### **Интеракции (Flows):**  
1. **Пользователь → Платформа:**  
   - Запрос на поиск ресторанов, блюд, оформление заказа, отслеживание доставки, отзыв.  
2. **Платформа → Платежный шлюз:**  
   - Запрос на обработку платежа (сумма, курьер, ресторан).  
3. **Платежный шлюз → Платформа:**  
   - Уведомление о статусе платежа (успешно/отклонено).  
4. **Платформа → Ресторан:**  
   - Уведомление о новом заказе, запрос на обновление статуса готовности.  
5. **Ресторан → Платформа:**  
   - Обновление статуса заказа (готов/отправлен).  
6. **Платформа → Курьер:**  
   - Назначение заказа, запрос на обновление статуса доставки, маршрутизация.  
7. **Курьер → Платформа:**  
   - Обновление статуса доставки (в пути, доставлено), геолокация.  
8. **Платформа → Сервис карт:**  
   - Запрос маршрута (адрес ресторана → адрес пользователя).  
9. **Сервис карт → Платформа:**  
   - Возврат маршрута, расстояния, времени.  
10. **Пользователь → Система поддержки:**  
   - Запрос помощи, жалобы, запросы на возврат.  
11. **Система поддержки → Платформа:**  
   - Обновление статуса запроса, уведомление пользователя.
