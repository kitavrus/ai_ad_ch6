# Project Title
ИИ CLI чат с сохранением диалога

## Overview
Эта программа реализует интерфейс командной строки (CLI) для общения с API, совместимым с OpenAI. Она сохраняет историю диалога и переписки на диск и пишет per‑request метрики в отдельные логи. Поддерживается возобновление сессии и продолжение диалога с учётом контекста.

## Цели
- Предоставить надёжный интерактивный CLI для чата с моделью ИИ.
- Сохранять одну сессию как единый JSON-файл и постепенно обновлять его во время диалога.
- Создавать per‑request логи метрик в отдельной директории, по одному файлу на запрос.
- Предусмотреть возможность возобновления последней сессии и продолжения диалога.
- Обеспечить чистую и хорошо документированную кодовую базу, соответствующую PEP 8.

## Архитектура

### CLI Frontend (`script.py`)
- Разбор аргументов командной строки: модель, URL API, параметры генерации, промпты и флаг возобновления.
- Формирует начальные системный и пользовательский промпты, затем входит в интерактивный цикл.
- Принимает ввод пользователя, добавляет его к диалогу, отправляет запрос к API и выводит ответ ассистента, обновляя сессию.
- При выходе сохраняет финальную сессию.

### API Client Interaction
- Использует OpenAI‑совместимый клиент для отправки запросов chat completions.
- Поддерживает дополнительные поля, например `top_k`, через аргумент `extra_body`.

### Persistence Layer (слой сохранения данных)
- Сессия: `dialogues/session_<timestamp>_<model>.json` (один файл на запуск; перезаписывается)
- Метрики per-request: `dialogues/metrics/session_<session_id>_req_<idx>.log` (JSON)

### Метаданные и журналирование
- Метрики: TTFT, ReqTime, TotalTime, Tokens, p_tokens, c_tokens, Cost
- per-request логи — JSON с перечисленными полями

### Поддержка возобновления
- CLI‑флаг `--resume` позволяет загрузить последнюю сохранённую сессию и продолжить диалог
- Inline-команда `/resume true` позволяет возобновить сессию прямо в процессе диалога

### Расширяемость
- Легко разделить на модули (`api_client`, `persistence`, `cli`) и добавить тесты

## Структура модуля `script.py`

Файл разделён на логические секции:

| Секция | Содержимое |
|--------|-----------|
| Константы | `BASE_URL`, `API_KEY`, `DEFAULT_*`, `USD_PER_1K_TOKENS`, `RUB_PER_USD` |
| Вспомогательные функции | `parse_args`, `_parse_inline_command`, `_get_usage_value`, `_print_loaded_history`, `_apply_session_data`, `_apply_updates` |
| Слой сохранения данных | `_save_session_to_path`, `_log_request_metric`, `_load_last_session` |
| Основная функция | `main` |

### Публичные функции и их назначение

- **`parse_args() -> argparse.Namespace`** — разбор аргументов командной строки.
- **`main() -> None`** — основной поток выполнения: инициализация, интерактивный цикл, сохранение сессии.

### Вспомогательные (приватные) функции

- **`_parse_inline_command(line) -> dict`** — разбор inline-команд вида `/ключ значение` или `/ключ=значение`.
- **`_get_usage_value(usage, key)`** — безопасное извлечение поля из объекта или словаря usage.
- **`_print_loaded_history(messages)`** — вывод последних 5 обменов из загруженной истории.
- **`_apply_session_data(data, *, model, base_url, system_prompt, initial_prompt) -> dict`** — применяет данные загруженной сессии, возвращает обновлённые значения.
- **`_apply_updates(updates, *, ...) -> dict`** — применяет inline-команды к конфигурации сессии, возвращает новое состояние.
- **`_save_session_to_path(session, path) -> str`** — сохраняет сессию в JSON-файл по указанному пути.
- **`_log_request_metric(req_entry, session_id, idx) -> str`** — сохраняет метаданные одного запроса в лог-файл.
- **`_load_last_session() -> Optional[Tuple[str, dict]]`** — загружает последнюю сессию из `dialogues/session_*.json`.

## CLI интерфейс

### Аргументы командной строки

| Аргумент | Короткий | По умолчанию | Описание |
|----------|----------|-------------|----------|
| `--model` | `-m` | `DEFAULT_MODEL` | Модель для использования |
| `--base-url` | `-u` | `BASE_URL` | Базовый URL API |
| `--max-tokens` | — | `None` | Максимальное число токенов в ответе |
| `--temperature` | `-T` | `0.7` | Температура генерации |
| `--top-p` | `-p` | `0.9` | Nucleus sampling |
| `--top-k` | `-k` | `50` | Top-k |
| `--system-prompt` | — | `None` | Системный промпт |
| `--initial-prompt` | — | `None` | Начальное сообщение пользователя (seed) |
| `--resume` | — | `False` | Загрузить последнюю сессию и продолжить диалог |

### Inline-команды (в процессе диалога)

Команды вводятся прямо в чате и начинаются с `/`:

```
/model=gpt-4
/base-url https://api.example.com
/max-tokens 1024
/temperature 0.8
/top-p 0.92
/top-k 50
/system-prompt Новый системный промпт
/initial-prompt Начальное сообщение
/resume true
```

## Использование

```bash
# Начать новую сессию
python script.py

# Возобновить последнюю сессию
python script.py --resume

# Настроить модель и параметры
python script.py -m "qwen/qwen3.5-plus-02-15" -T 0.8

# Задать системный промпт
python script.py --system-prompt "Ты помощник-программист"
```

Логи метрик сохраняются автоматически в `dialogues/metrics/`.

## Что делает код (высокоуровнево)

1. Инициализация клиента OpenAI с API‑ключом и базовым URL.
2. Формирование начальных сообщений (`system_prompt`, `initial_prompt`).
3. Вход в интерактивный цикл:
   - Чтение ввода пользователя.
   - Обработка inline-команд (`/команда`).
   - Добавление сообщения в историю с ролью `"user"`.
   - Отправка запроса к API (chat completions).
   - Вывод ответа ассистента.
   - Добавление ответа в историю с ролью `"assistant"`.
   - Сохранение per-request метрики в `dialogues/metrics/`.
   - Обновление файла сессии `dialogues/session_*.json`.
4. По выходу (`exit` / `quit` / `q` / Ctrl+C) — сохранение финального состояния сессии.

## Изменения рефакторинга (PEP 8, 2026-02-25)

В ходе рефакторинга по стандартам PEP 8 были выполнены следующие изменения:

- **Импорты** — все `import` перенесены в начало файла; устранён двойной импорт `import os, json` в одной строке (E401/E402).
- **Константы** — `USD_PER_1K_TOKENS` и `RUB_PER_USD` вынесены из тела цикла на уровень модуля.
- **Дублирующиеся комментарии** — удалены повторяющиеся строки комментариев.
- **Дублирующиеся ключи** — исправлены дублирующиеся ключи в `elif`-ветках `_parse_inline_command`.
- **Антипаттерны** — убраны `"session_path" not in locals()` и избыточный `try: messages except NameError`.
- **Неиспользуемые переменные** — удалена `resume_used = False` (F841).
- **Мёртвый код** — удалены неиспользуемые функции `_save_dialogue()` и `_save_session()`.
- **Вынос вложенной функции** — `_apply_updates()` вынесена из тела `main()` как функция модуля; логика загрузки сессии выделена в `_apply_session_data()` и `_print_loaded_history()`.
- **Docstring** — добавлены к `parse_args()` и `main()`; все docstring приведены к стилю PEP 257.
- **Комментарии** — унифицированы на русском языке; закомментированный код убран.
- **Структура файла** — добавлены секционные разделители (`# ---`) для констант, вспомогательных функций, слоя сохранения и основной функции.
- **Обработка исключений** — `except Exception` заменён на конкретные `except (ValueError, TypeError)` там, где это применимо.

Этот файл самодостаточен и описывает требования, структуру функций и формат файлов. Его можно использовать как единый паттерн для генерации кода. При генерации можно расширять и создавать отдельные модули: `api_client.py`, `persistence.py`, `cli.py`.
