# CRM — Руководство по работе с задачами

## Жизненный цикл задачи

Каждая задача в CRM проходит три статуса:

```
NEW  →  IN_PROCESS  →  DONE
```

- **NEW** — задача создана, исполнитель ещё не приступил
- **IN_PROCESS** — задача взята в работу, исполнитель активно её выполняет
- **DONE** — задача завершена, результат достигнут

Переход **NEW → IN_PROCESS** означает принятие задачи в работу. Исполнитель обязан сменить статус до начала активных действий по задаче.

---

## Приоритеты

| Приоритет | Значение | Срок реакции |
|-----------|----------|--------------|
| **HIGH**  | Критично для клиента или сделки | В течение сегодняшнего дня |
| **MEDIUM**| Плановая активность | В течение 2–3 рабочих дней |
| **LOW**   | Административная задача | В течение недели |

## Типы задач и инструкции по выполнению

### Связаться с клиентом (звонок / письмо)

**Когда возникает:** менеджер должен перезвонить или написать клиенту по вопросу договора, сделки или поддержки.

**Шаги выполнения:**
1. Изучить карточку клиента в CRM — историю переговоров, открытые сделки.
2. Подготовить повестку звонка: цель, ключевые вопросы, ожидаемый результат.
3. Связаться с клиентом (звонок или email).
4. Зафиксировать итог переговоров в описании задачи.
5. Если нужно продолжение — создать новую задачу со статусом NEW.
6. Сменить статус задачи на DONE.

---

### Подготовить коммерческое предложение (КП)

**Когда возникает:** клиент заинтересован в продукте, нужно подготовить предложение с условиями и ценами.

**Шаги выполнения:**
1. Уточнить у менеджера требования клиента и бюджет.
2. Подготовить КП по шаблону компании: описание решения, состав, цены, сроки.
3. Согласовать КП с руководителем (если сумма выше порогового значения).
4. Отправить КП клиенту по email, зафиксировать дату отправки.
5. Поставить задачу на follow-up через 2–3 дня.
6. Сменить статус задачи на DONE.

---

### Провести демо продукта

**Когда возникает:** клиент хочет увидеть продукт в действии перед принятием решения.

**Шаги выполнения:**
1. Согласовать дату и формат демо (онлайн / офлайн) с клиентом.
2. Подготовить демо-стенд и сценарий показа под задачи клиента.
3. Провести демо, зафиксировать вопросы и возражения клиента.
4. Отправить материалы презентации после встречи.
5. Создать задачу на подготовку КП или следующий шаг.
6. Сменить статус задачи на DONE.

---

### Обновить CRM-карточки клиентов

**Когда возникает:** плановая актуализация данных — контактные телефоны, email, ответственные лица.

**Шаги выполнения:**
1. Открыть список клиентов за указанный период (например, Q1).
2. Проверить и обновить: ФИО контактного лица, телефон, email, статус сделки.
3. Убрать дублирующиеся карточки.
4. Проставить актуальных ответственных менеджеров.
5. Сменить статус задачи на DONE.

---

### Закрыть сделку

**Когда возникает:** клиент готов подписать договор и произвести оплату.

**Шаги выполнения:**
1. Подготовить договор по шаблону юридического отдела.
2. Согласовать условия с клиентом (скидки, этапы оплаты, сроки).
3. Подписать договор — электронная или живая подпись.
4. Выставить счёт на оплату.
5. Убедиться, что оплата получена (или поставить задачу на контроль оплаты).
6. Сменить статус задачи на DONE.

---

## Роли и ответственность

| Роль | Зоны ответственности |
|------|---------------------|
| **Менеджер** | Работа с клиентами, переговоры, звонки, КП |
| **Исполнитель** | Выполнение задачи, смена статусов, фиксация результата |
| **Руководитель** | Контроль HIGH-приоритетных задач, согласование крупных сделок |

---

## Частые ошибки

- Не менять статус при взятии задачи в работу — коллеги не видят, кто чем занят.
- Закрывать задачу без описания результата — теряется история.
- Создавать дублирующие задачи вместо обновления существующих.
