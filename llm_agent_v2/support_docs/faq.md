# FAQ — Поддержка продукта

## Авторизация

### Почему я получаю ошибку 401 Unauthorized?
Ошибка 401 означает, что ваш токен доступа недействителен или истёк. Проверьте следующее:
- Токен передаётся в заголовке `Authorization: Bearer <token>`
- Срок действия токена не истёк (по умолчанию 1 час)
- Токен был выдан для текущего окружения (prod/staging)
- Для обновления токена используйте refresh-token через `/auth/refresh`

### Почему я получаю ошибку 403 Forbidden?
Ошибка 403 означает, что у вашего аккаунта нет прав на данное действие:
- Убедитесь, что ваш тариф включает нужную функцию
- Проверьте, что токен имеет нужные scopes (например, `write:data`)
- Для enterprise-функций требуется тариф Enterprise или выше

### Как настроить OAuth авторизацию?
1. Создайте OAuth-приложение в настройках аккаунта → «Интеграции»
2. Укажите `redirect_uri` — он должен точно совпадать с зарегистрированным
3. Используйте Authorization Code Flow для серверных приложений
4. Получите `code`, обменяйте на `access_token` через POST `/oauth/token`
5. Refresh token действует 30 дней, access token — 1 час

### Письмо для сброса пароля не приходит
- Проверьте папку «Спам» и «Промоакции»
- Письмо приходит в течение 5 минут
- Убедитесь, что вводите email, привязанный к аккаунту
- Если письмо не пришло через 10 минут — создайте тикет в поддержке

### Как включить двухфакторную аутентификацию (2FA)?
1. Перейдите в Настройки → Безопасность → Двухфакторная аутентификация
2. Установите приложение-аутентификатор (Google Authenticator, Authy)
3. Отсканируйте QR-код или введите секретный ключ вручную
4. Введите 6-значный код для подтверждения
5. Сохраните резервные коды в безопасном месте

## Billing и тарифы

### Какие тарифы доступны?
- **Free**: до 100 запросов/день, базовые функции, без SLA
- **Pro**: до 1000 req/min, приоритетная поддержка, 99.9% SLA, вебхуки
- **Enterprise**: неограниченные запросы, выделенный менеджер, 99.99% SLA, кастомные интеграции

### Почему моя карта была отклонена (CARD_DECLINED)?
Возможные причины:
- Недостаточно средств на карте
- Банк блокирует международные транзакции — обратитесь в банк
- Истёк срок действия карты
- Включена защита от мошенничества — разрешите транзакции от нашего сервиса
- Попробуйте другую карту или PayPal через Настройки → Оплата

### Как изменить тариф?
Перейдите в Настройки → Подписка → «Изменить тариф». Апгрейд активируется немедленно, даунгрейд — с начала следующего расчётного периода.

### Что произойдёт при превышении лимитов на Free-тарифе?
Запросы сверх лимита будут возвращать ошибку `429 Too Many Requests`. Вы можете:
- Подождать сброса лимита (в полночь UTC)
- Перейти на тариф Pro для получения более высоких лимитов

## API

### Какие лимиты запросов (rate limits)?
| Тариф | Лимит |
|-------|-------|
| Free | 100 req/day |
| Pro | 1000 req/min |
| Enterprise | без лимитов (fair use) |

При превышении лимита API возвращает `429 Too Many Requests` с заголовком `Retry-After: <seconds>`.

### Как настроить Webhook?
1. Перейдите в Настройки → Интеграции → Webhooks → «Добавить endpoint»
2. Укажите HTTPS URL (HTTP не поддерживается)
3. Выберите события: `order.created`, `payment.success`, `user.updated` и др.
4. Скопируйте `Signing Secret` для верификации подписи
5. Каждый запрос содержит заголовок `X-Signature-256` — проверяйте его
6. Endpoint должен отвечать `200 OK` в течение 5 секунд

### Webhook не получает события
Проверьте:
- URL доступен публично (не localhost)
- Порт открыт для входящих соединений
- Endpoint возвращает `200 OK` (не редирект)
- В логах Webhook (Настройки → Интеграции → Webhooks → «Логи») есть попытки доставки
- Проверьте верификацию подписи — неверная подпись блокирует обработку

### Как получить список доступных методов API?
Полная документация доступна в разделе «Документация» → «API Reference». Используйте `/api/v1/schema` для получения OpenAPI-спецификации.

## Аккаунт

### Как изменить email аккаунта?
Перейдите в Настройки → Профиль → «Изменить email». На новый адрес придёт письмо для подтверждения. Старый email остаётся активным до подтверждения нового.

### Как удалить аккаунт?
Настройки → Профиль → «Удалить аккаунт». Удаление необратимо. Все данные будут удалены через 30 дней (можно отменить в течение этого срока). При наличии активной подписки — сначала отмените её.
