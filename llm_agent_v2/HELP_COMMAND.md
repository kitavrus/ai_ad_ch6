## Изоляция от истории диалога

`/help` не попадает в основную историю. После него можно продолжать обычный диалог:

```
Вы: /help как устроен RAG?
[Help] ...ответ...

Вы: напиши привет
Ассистент: Привет!    ← обычный режим, контекст /help не виден
```

---

## Ошибки и edge cases

```
Вы: /help          ← пустой вопрос
[Укажите вопрос: /help <вопрос>]

Вы: /helpкакой-то текст   ← нет пробела после /help — обрабатывается как обычное сообщение
```

---

## Что происходит внутри при каждом /help

1. RAG: поиск top-3 чанков по `structure`-индексу (`rag_index/structure.faiss`)
2. MCP: вызов `get_current_branch` + `list_files` через `git_server.py`
3. LLM: запрос с системным промптом (роль + RAG + git), без tool-calls
4. Вывод: `[Help] <ответ>`
