# RAG + История диалога + Память задачи — Руководство по ручному тестированию

## Что реализовано

- **RAG + история диалога** — контекст сохраняется в `session_*.json`
- **Память задачи (working memory)** — цель, ограничения, уточнения
- **Профильная изоляция** — `dialogues/{profile}/`
- `--profile NAME` автоматически загружает последнюю сессию профиля
- Без профиля (default) используйте команду `/resume` для загрузки старой сессии

---

## Шаг 1 — Запуск БЕЗ RAG (baseline)

```bash
cd ai_ad_ch6 && source .venv/bin/activate && cd llm_agent
python script.py
```

1. Задать 2 вопроса:
   - `Сколько уровней памяти в этом чатботе?`
   - `Что такое стратегия sliding_window?`
2. Убедиться что ответы **не содержат** секцию `**Источники:**`.
3. Выполнить `/rag status` — должен ответить `[RAG выключен]`.
4. Выйти (`exit`).

**Ожидаемый результат:**

| Проверка | Ожидание |
|----------|----------|
| Секция `**Источники:**` в ответах | Отсутствует |
| `/rag status` | `[RAG выключен]` |

**Чеклист:**
- ☐ Ответы без `**Источники:**`
- ☐ `/rag status` = выключен

---

## Шаг 2 — Запуск С RAG

```bash
python script.py
/rag on
/rag status
```

1. Задать 2 вопроса:
   - `Как устроена трёхуровневая система памяти?`
   - `Как профильная изоляция реализована в хранилище?`
2. Проверить что ответы содержат `**Ответ:**`, `**Источники:**`, `**Цитаты:**`.
3. В консоли должна появиться строка `[RAG] Найдено N чанк(ов) из: ...`.

**Ожидаемый результат:**

| Проверка | Ожидание |
|----------|----------|
| `/rag status` | `[RAG включён]` |
| `**Источники:**` в ответах | Присутствует |
| Сообщение в консоли | `[RAG] Найдено N чанк(ов) из: ...` |

**Чеклист:**
- ☐ RAG включён
- ☐ `**Источники:**` присутствует
- ☐ Консоль показывает найденные чанки

## Шаг 3 — Сохранение и восстановление сессии (/resume)

```bash
# Сессия 1: запустить, включить RAG, задать вопросы, выйти
python script.py
/rag on
```

Задать 3 вопроса:
- `Что такое рабочая память (working memory) в этом проекте?`
- `Как переключаться между профилями пользователей?`
- `Чем стратегия sticky_facts отличается от branching?`

Затем:

```bash
exit
```

```bash
# Сессия 2: запустить заново и восстановить последнюю сессию default-профиля
python script.py
/resume
/rag status          # должен быть ON
```

Задать продолжающий вопрос, который ссылается на ответ из сессии 1:
- `Ты упоминал стратегию sticky_facts — как в ней хранятся факты между сессиями?`

**Ожидаемый результат:**

| Проверка | Ожидание |
|----------|----------|
| `/resume` | Загружает историю предыдущей сессии |
| `/rag status` после `/resume` | `[RAG включён]` |
| Ответ на продолжающий вопрос | Учитывает контекст сессии 1 |

**Чеклист:**
- ☐ `/resume` загружает историю
- ☐ RAG ON сохраняется между сессиями
- ☐ Контекст диалога не потерян

## Шаг 5 — Память задачи (/settask + /memshow)

```bash
python script.py --profile Test
/rag on
/settask "Понять как работает RAG в этом проекте"
```

Задать 5 вопросов, постепенно уточняя тему:

```
1. Какие стратегии RAG поддерживаются?
2. Что такое параметр top_k?
3. Как работает IDK-режим?
4. Какие форматы документов поддерживает RAG?
5. Как RAG взаимодействует с историей диалога?
```

Затем:

```bash
/memshow             # проверить working memory
/remember цель="разобраться с RAG"
/memshow
exit
```

```bash
python script.py --profile Test   # auto-resume
/memshow             # цель должна сохраниться
```

**Ожидаемый результат:**

| Проверка | Ожидание |
|----------|----------|
| `/memshow` после 5 вопросов | Working memory содержит накопленный контекст |
| `/remember` | Добавляет запись в working memory |
| `/memshow` после auto-resume | Цель и контекст сохранены |
| Ответы с учётом задачи | Ассистент ориентируется на задачу из `/settask` |

**Чеклист:**
- ☐ Задача сохраняется в working memory
- ☐ Working memory восстанавливается после auto-resume
- ☐ Ответы учитывают контекст задачи
